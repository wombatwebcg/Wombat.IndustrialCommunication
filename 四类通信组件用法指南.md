# 四类通信组件用法指南（跨项目接入）

## 1. 适用范围

本文面向外部项目，说明以下四个组件的定位、启动方式、读写方式与接入注意事项：

- `ModbusRtuClient`
- `ModbusRtuServer`
- `ModbusTcpServer`
- `S7TcpServer`

---

## 2. 快速结论

- `ModbusRtuClient` 是主站客户端，负责主动连设备并读写。
- `ModbusRtuServer`、`ModbusTcpServer` 是从站服务器，核心是维护 `DataStore`，不建议直接调用其 IReadWrite 风格读写接口。
- `S7TcpServer` 是可模拟西门子 PLC 的服务器，既可走协议响应，也提供本地内存区直接读写接口。
- 三个 Server 都是 `Listen()/Shutdown()` 生命周期；`ModbusRtuClient` 是 `Connect()/Disconnect()` 生命周期。

---

## 3. ModbusRtuClient 用法

### 3.1 构造与连接

```csharp
using Wombat.IndustrialCommunication.Modbus;
using System.IO.Ports;

var client = new ModbusRtuClient(
    portName: "COM3",
    baudRate: 9600,
    dataBits: 8,
    stopBits: StopBits.One,
    parity: Parity.None,
    handshake: Handshake.None);

client.IsLongConnection = true;
client.EnableAutoReconnect = true;
client.MaxReconnectAttempts = 5;
client.ReconnectDelay = TimeSpan.FromSeconds(2);

var connectResult = client.Connect();
if (!connectResult.IsSuccess) throw new Exception(connectResult.Message);
```

### 3.2 同步读写（推荐先掌握）

```csharp
byte station = 1;

var writeCoil = client.WriteCoil(station, 0, true);
var readCoil = client.ReadCoil(station, 0);

var writeReg = client.WriteHoldingRegister(station, 0, 1234);
var readReg = client.ReadHoldingRegister(station, 0);
```

### 3.3 异步读写

```csharp
await client.ConnectAsync();
await client.WriteCoilAsync(1, 0, true);
var coil = await client.ReadCoilAsync(1, 0);
await client.DisconnectAsync();
```

### 3.4 批量读写

```csharp
using Wombat.Extensions.DataTypeExtensions;

var writeMap = new Dictionary<string, (DataTypeEnums, object)>
{
    { "1;5;0", (DataTypeEnums.Bool, true) },
    { "1;6;1", (DataTypeEnums.UInt16, (ushort)100) }
};
await client.BatchWriteAsync(writeMap);

var readMap = new Dictionary<string, DataTypeEnums>
{
    { "1;1;0", DataTypeEnums.Bool },
    { "1;3;1", DataTypeEnums.UInt16 }
};
var batch = await client.BatchReadAsync(readMap);
```

### 3.5 地址格式

支持两类格式：

- 标准格式：`站号;功能码;地址`，例如 `1;3;0`
- 增强格式：`站号;地址`，例如：
  - `1;40001`（保持寄存器，映射到协议地址 0，自动推断功能码）
  - `1;30001`（输入寄存器）
  - `1;10001`（线圈）

## 4. ModbusRtuServer 用法

### 4.1 构造与监听

```csharp
using Wombat.IndustrialCommunication.Modbus;
using System.IO.Ports;

var server = new ModbusRtuServer("COM3", 9600, 8, StopBits.One, Parity.None, Handshake.None);
var listen = server.Listen();
if (!listen.IsSuccess) throw new Exception(listen.Message);
```

### 4.2 核心操作：维护 DataStore

`ModbusRtuServer` 的职责是响应来自客户端的 Modbus 功能码请求，读写数据来源于 `server.DataStore`：

```csharp
server.DataStore.CoilDiscretes[0] = true;
server.DataStore.InputDiscretes[0] = false;
server.DataStore.HoldingRegisters[0] = 1234;
server.DataStore.InputRegisters[0] = 5678;
```

### 4.3 重要注意

- 该类实现了大量 `ReadXxx/WriteXxx` 接口，但会返回“不支持此操作”，这是设计行为。
- 服务器端正确用法是维护 `DataStore`，让外部 Modbus 客户端通过协议访问。

---

## 5. ModbusTcpServer 用法

### 5.1 构造与监听

```csharp
using Wombat.IndustrialCommunication.Modbus;

var server = new ModbusTcpServer("0.0.0.0", 502);
server.MaxConnections = 200;
var listen = server.Listen();
if (!listen.IsSuccess) throw new Exception(listen.Message);
```

### 5.2 维护 DataStore

与 RTU 服务器一致，通过 `DataStore` 承载业务数据：

```csharp
server.DataStore.CoilDiscretes[0] = true;
server.DataStore.HoldingRegisters[0] = 1000;
```

### 5.3 重要注意

- 服务器类上的通用 `Read/Write/BatchRead/BatchWrite` 接口为“不支持”返回，属于保护性实现。
- 业务上应将其视为“协议网关 + 内存镜像”。

---

## 6. S7TcpServer 用法

### 6.1 构造、PLC 参数、监听

```csharp
using Wombat.IndustrialCommunication.PLC;

var s7 = new S7TcpServer("0.0.0.0", 102);
s7.SetSiemensVersion(SiemensVersion.S7_1200);
s7.SetRackSlot(0, 1);
s7.CreateDataBlock(1, 1024);

var listen = s7.Listen();
if (!listen.IsSuccess) throw new Exception(listen.Message);
```

### 6.2 直接按区域操作

```csharp
s7.WriteDB(1, 0, new byte[] { 0x00, 0x64 });
var dbBytes = s7.ReadDB(1, 0, 2);

s7.WriteMerkers(0, new byte[] { 1, 2, 3, 4 });
var mBytes = s7.ReadMerkers(0, 4);
```

### 6.3 按地址字符串读写（IReadWrite 风格）

```csharp
using Wombat.Extensions.DataTypeExtensions;

s7.Write("DB1.DBW0", (ushort)32000);
var w = s7.ReadUInt16("DB1.DBW0");

s7.Write("DB1.DBX2.0", true);
var x = s7.ReadBoolean("DB1.DBX2.0");

s7.Write("M10", (byte)0x12);
var m = s7.ReadByte("M10");

s7.Write("V100", (byte)0x34);
var v = s7.ReadByte("V100");
```

### 6.4 地址规则

- DB 区：`DB1.DBX0.0` / `DB1.DBB0` / `DB1.DBW0` / `DB1.DBD0`
- M 区：`M0`
- I 区：`I0`
- Q 区：`Q0`
- V 区：`V100`（内部映射到 `DB1`）

### 6.5 数据变化事件

```csharp
s7.DataWritten += (_, e) =>
{
    Console.WriteLine($"{e.Area} {e.StartAddress} {e.Data?.Length}");
};

s7.DataRead += (_, e) =>
{
    Console.WriteLine($"{e.Area} {e.StartAddress} {e.Data?.Length}");
};
```

---

## 7. 生命周期建议

- `ModbusRtuClient`：`Connect -> Read/Write -> Disconnect`
- `ModbusRtuServer`：`Listen -> 维护 DataStore -> Shutdown`
- `ModbusTcpServer`：`Listen -> 维护 DataStore -> Shutdown`
- `S7TcpServer`：`CreateDataBlock/初始化区域 -> Listen -> Shutdown`

---

## 8. 跨项目接入最小模板

```csharp
using Wombat.IndustrialCommunication.Modbus;
using Wombat.IndustrialCommunication.PLC;
using Wombat.Extensions.DataTypeExtensions;
```

- Modbus 主站采集：优先使用 `ModbusRtuClient`
- Modbus 从站模拟：使用 `ModbusRtuServer` 或 `ModbusTcpServer` 并维护 `DataStore`
- 西门子 PLC 模拟：使用 `S7TcpServer`，按 DB/M/I/Q/T/C 区组织数据

---

## 9. 停止与释放

```csharp
client?.Disconnect();
modbusRtuServer?.Shutdown();
modbusTcpServer?.Shutdown();
s7?.Shutdown();
```

建议在宿主进程退出时统一执行停止，避免端口或串口资源残留占用。
